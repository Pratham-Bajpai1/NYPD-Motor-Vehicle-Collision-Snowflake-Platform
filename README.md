# 🚔 NYPD Motor Vehicle Collisions Analytics Pipeline

## 📌 Project Overview

An end-to-end cloud data engineering pipeline built using **Snowflake Data Cloud** and **AWS S3**. This project demonstrates the implementation of a modern Medallion Architecture to ingest, cleanse, secure, and share 1.6M+ records of NYPD Motor Vehicle Collision data.

### Problem Statement & Objectives

The NYPD provides massive datasets on motor vehicle traffic accidents across New York City. However, a significant portion of historical records are plagued by dirty and missing geographical data (Borough and Zip Codes).

**Objectives:**

* Secure Ingestion: Establish a secure cloud-to-cloud connection between AWS S3 and Snowflake without exposing secret credentials.

* Geospatial Data Cleansing: Build an algorithmic lookup to recover missing fields by analyzing localized coordinate patterns.

* Advanced Analytics: Produce critical operational insights regarding accident frequencies, high-risk boroughs, and prominent vehicle types.

* Data Governance & Masking: Protect sensitive citizen registration data based on organizational role permissions.

* Data Sharing: Securely distribute live-updating query results to non-Snowflake third parties using zero-copy cloning principles.

The project follows modern cloud data engineering practices and demonstrates Snowflake's capabilities for scalable analytics, governance, and data sharing.

# 🧹 Data Cleansing

## Challenge

The dataset contained missing:

* Borough
* ZIP Code

information for a large number of records.

### Initial Missing Records

| Metric                   | Count   |
| ------------------------ | ------- |
| Missing Borough/ZIP Code | 484,625 |

---

## Cleansing Strategy

A geolocation mapping table was created using:

```sql
TRUNC(LATITUDE)
TRUNC(LONGITUDE)
```

For each coordinate block:

* Most frequently occurring Borough selected
* Most frequently occurring ZIP Code selected

using:

```sql
ROW_NUMBER()
COUNT()
QUALIFY
```

### Rules Applied

* Ignored NULL latitude/longitude
* Ignored invalid 0,0 coordinates
* Reconstructed missing Borough values
* Reconstructed missing ZIP Code values
